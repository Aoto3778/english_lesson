# 朗読コーチ — Oral Reading Coach

英文を朗読し、**発音・イントネーション・強勢(抑揚)・間の取り方・速さ**について
まとめてフィードバックを受けられる、ブラウザ完結の英語朗読コーチングアプリです。

`index.html` 単体で動作します。ビルド不要・**完全無料**・サーバー不要。

## 特徴

- **完全クライアントサイド処理** — 録音・解析(ピッチ/間/速さ)はすべて端末内で完結し、
  音声は保存されません。発音判定のときのみ、ブラウザ内蔵の音声認識へ音声が送られます。
- **使用技術はすべてブラウザ標準・無料**
  - マイク録音: `getUserMedia` + `MediaRecorder`
  - 音声解析: Web Audio API(`decodeAudioData` で取得した PCM を端末内で解析)
    - フレーム化(窓50ms/ホップ25ms)+ RMS、ノイズフロア推定による無音・「間」検出
    - 自己相関法による F0(ピッチ)抽出 → イントネーション曲線
  - お手本音声: `speechSynthesis`(en-US)
  - 発音の正誤判定: `SpeechRecognition` + 原文との単語アラインメント(動的計画法)
  - データ保存: `localStorage`(履歴・設定。最大30件、音声は保存しない)
- **4画面**: テキスト入力 / 朗読(録音)/ フィードバック結果 / 練習履歴
- **グレースフルデグレード**: `SpeechRecognition` 非対応(iPad Safari 等)やオフラインでも、
  発音以外(ピッチ・間・速さ)で結果が成立します。

## 使い方 / 配信

`getUserMedia` の都合上、**HTTPS(または `localhost`)で配信**する必要があります。

ローカル確認:

```bash
# このディレクトリで簡易サーバを起動し、http://localhost:8000 を開く
python3 -m http.server 8000
```

公開する場合は GitHub Pages など HTTPS の静的ホスティングに `index.html` を置くだけです。

### 対応環境

| ブラウザ | 状態 |
|---|---|
| Chrome / Edge(Windows）| フル機能(発音判定を含む) |
| iPad Safari | 発音判定以外フル機能(端末内・オフライン可) |

## 既知の限界

- 発音判定は音声認識ベースの**近似**であり、音素レベルの厳密評価ではありません。
- `SpeechRecognition` は Chrome / Edge かつネット接続時のみ動作します。
- TTS(お手本)の音質・話者は OS 依存です。en-US 音声を優先選択します。
